# skill-manager constitution

这是 `skill-manager` 的最高优先级原则。任何 AI 接手时，先看这份。

## 1. 命令定义必须固定

- `scan`：在本地扫描所有技能库
- `steal`：从其他技能库迁移到这里
- `check`：默认检查当前库健康度
- `act`：联网后按当前身份推荐 skills

不要发明新的命令心智。

## 2. 先信脚本，再信总结

- 脚本输出是事实源。
- 后续自然语言总结只能压缩、翻译、排序。
- 不能重算数字，不能补不存在的状态，不能改写命令语义。

## 3. 先信当前项目，再信主目录

- 最近的项目级 `.claude/skills`、`CLAUDE.md`、`AGENTS.md` 优先。
- 主目录是全局资源池，不等于当前项目已经生效。

## 4. 用户给路径时，先信路径

- 绝对路径是强输入。
- 先验证这个路径本身。
- 不能跳去别的库乱找。

## 5. 状态必须和证据一致

- 失败就是失败。
- 超时就是超时。
- 静态通过不等于实机通过。

## 6. `check` 是主命令，`act` 是轻量在线视图

- `check` 负责当前库 / 路线判断。
- `act` 负责身份驱动的在线推荐。
- `act` 不能膨胀成第二套完整体检。

## 7. 在线推荐先看匹配，再看热度

- 搜到不等于推荐。
- 低星候选优先当灵感。
- 没有明显高热结果时，优先本地库和官方入口。

## 8. Windows 兼容是增量兼容

- 不能为了兼容 Windows 破坏 macOS 的本地扫描优势。
- 优先保留当前扫描能力，再补平台差异。

## 9. 每次迭代都要落到文件里

- 新坑，写进 `gotchas.md`
- 新边界，写进 `output-contract.md`
- 新判断流程，写进 checklist
- 新总原则，先考虑是否要更新这份 `constitution.md`
