# check-current checklist

`check` 默认按这份清单判断当前库，不要跳步。

## A. 目标确认

- 当前目标库是谁
- 是项目级，还是主目录级
- 当前宿主是谁

## B. 结构检查

- 是否有缺失 `SKILL.md` 的目录
- 是否有 frontmatter 缺失或明显异常
- 是否有断开的软链
- 是否有同名或近似重复技能

## C. 宿主兼容

- 是否引用了其他宿主的目录约定
- 是否依赖特定的 host 文件
- 是否需要项目级规则文件配合

## D. 依赖与运行条件

- 是否缺 CLI 工具
- 是否缺 API key / token
- 是否需要本地服务或端口
- 是否只做了静态检查，还没做实机验证

## E. 输出格式

必须按这个顺序：

1. 当前库是谁
2. 整体状态
3. 发现的问题
4. 下一步建议

## F. 用词约束

- 用“当前看起来可用”，不要用“完全没问题”
- 用“需配置 / 需用户操作”，不要用“废技能”
- 如果只是警告，要说清这是不是阻塞
