# skill-manager gotchas

这份文件只记录高价值坑点。优先级高于“经验性发挥”。

## 1. 路径优先

- 用户给了绝对路径，就先查这个路径。
- 先说清它是项目级 skill，还是主目录 skill。
- 不能先跳去 `scan`、`grep`、别的库乱找。

## 2. 当前项目优先于主目录

- 最近的项目级 `.claude/skills`、`CLAUDE.md`、`AGENTS.md` 优先。
- 主目录 `~/.claude/skills` 是全局库，不等于项目里已经安装。
- 不要把“主目录存在同名 skill”说成“当前项目已经在用”。

## 3. 软链要单独看

- 软链存在，不等于目标一定有效。
- 需要区分：
  - 软链本身在不在
  - 指向目标在不在
  - 指向的是哪个来源库
- 不能把软链问题误判成“文件没找对”或“已经可用”。

## 4. 状态必须跟证据一致

- 命令失败、超时、缺文件，就不能说“已启动”“已连接”“搞定”。
- 只有探针成功，才能说 ready。
- 静态检查通过，只能说“当前看起来可用”，不能说“实机已验证”。

## 5. 不要重算脚本已经给出的数字

- 脚本说当前库 `14` 个技能，就直接复述 `14`。
- 不要自己再数一遍，然后数错。
- 不要把“实体数 + 软链数”讲乱。

## 6. 不要发明命令语义

- `scan`：在本地扫描所有技能库
- `steal`：从其他技能库迁移到这里
- `check`：默认检查当前库健康度
- `act`：联网后按当前身份推荐 skills

禁止：

- 把 `check` 说成检查单个 skill
- 把 `act` 说成第二套完整体检系统
- 把 `steal` 失败后的手工补丁伪装成“上游原版 skill”

## 7. 在线推荐先看匹配，再看热度

- `SkillsMP` 搜索命中不等于强推荐。
- 低星结果是灵感，不是首选安装。
- 没有明显高热结果时，优先建议用户先看本地大库和官方入口。

## 8. Windows 兼容不能牺牲 macOS 本地扫描

- Windows 兼容是增量兼容，不是替换扫描策略。
- macOS 仍要保留本地多来源扫描优势。
- 原生 Windows 更保守，优先复制而不是软链。
